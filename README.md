# cost-accounting-gas
属人化を排した原価計算フローの自動化

## 目的
- 原価計算の各工程を 「担当者単位の処理」 として分離
- 各処理は
  - 「入力 → 集計 → 出力（ファイルやシート）」
- 人が入れ替わっても処理は壊れないように

## 基本的想定（ver0~1）：評価方法など他基準へのは今後のバージョンアップで検討
- 費用の発生、費消の認識ごとに入力シートに記帳→月末一括処理
- 在庫評価：移動平均法（計算の簡便さ優先）
- 実際原価計算
- 1種の製品を製造している想定
- 当面、以下のように考える
  - 材料→直接費（仕掛品勘定へ）
  - 労務費→作業内容により直接費/間接費（仕掛品勘定/製造間接費勘定）
    - 勤怠が確定した時点で当月発生額を確定させる設計としている。（教科書的な期間のズレによる未払計算は想定しない。）
  - 経費→製造間接費
    - まずは単純で純粋な「原価計算機能」のみ。会計機能＝仕訳との関連を持たせることができれば間接費との配賦が可能になる。
