# 原価計算フロー再現ツール　cost-accounting-gas
本プロジェクトは、原価計算をスプレッドシートベースで整理・自動化したい中小規模事業者／学習者向けの設計検討用ツールです。

## 目的
- 原価計算の各工程を 「担当者単位の処理」 として分離
- 各処理は
  - 「入力 → 集計 → 出力（ファイルやシート）」
- 人が入れ替わっても処理は壊れないように

## 基本的想定（ver1）：評価方法など他基準へのは今後のバージョンアップで検討
- 費用の発生、費消の認識ごとに入力シートに記帳→月末一括処理
- 在庫評価：移動平均法（計算の簡便さ優先）
- 実際原価計算
- 1種の製品を製造している想定
- 当面、以下のように考える
  - 材料→直接費（仕掛品勘定へ）
  - 労務費→作業内容により直接費/間接費（仕掛品勘定/製造間接費勘定）
    - 勤怠が確定した時点で当月発生額を確定させる設計としている。
    - 教科書的な「賃金支払期間と原価計算期間のズレ」は、賃金支払管理の問題であり、原価計算に必須ではないと判断した。
  - 経費→製造間接費: v2以降
    - まずは単純で純粋な「原価計算機能」のみ。会計機能＝仕訳との関連を持たせることができれば間接費との配賦が可能になる。

## 使用方法
1. 材料受払簿、勤怠管理簿に相当するスプレッドシートを用意する。（詳細なシート構成はdesign.md参照）
2. .gsファイル群を導入
3. config.gsを利用環境に合わせる
4. 各シートの拡張機能→App Scriptで対応する関数を呼び出す設定を行う。<br>（ライブラリ設定→呼び出し、ツールバーメニュー作成）<br>※ カスタムメニューを利用して、ボタン操作のみで集計できる設計を想定しています。
5. 各ファイルを開くとツールバーに原価計算メニューが出てくるので、そこで集計する。
